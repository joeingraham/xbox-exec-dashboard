\U0001F4CB Methodology & Data Notes

Data Sources

\u2022 MAU/DAU: Xbox Live telemetry (trailing 30/1 day active sessions)
\u2022 Revenue: Finance data warehouse, FX-adjusted to USD
\u2022 Engagement: PlayFab session events, 99th percentile outliers excluded
\u2022 Retention: Cohort-based, measured from first session timestamp
\u2022 NPS: Quarterly survey (n=50K+ per franchise, margin \u00b11.2pp)
\u2022 Competitive: Third-party estimates (Newzoo, Sensor Tower, SteamDB)

Key Definitions

\u2022 MAU: Unique users with \u22651 authenticated session in trailing 30 days
\u2022 DAU/MAU Stickiness: Daily active users \u00f7 Monthly active users
\u2022 M1 Retention: % of new users with \u22651 session in days 31\u201360
\u2022 ARPU: Average Revenue Per User (total rev \u00f7 MAU)
\u2022 ARPPU: Average Revenue Per Paying User
\u2022 Gross Margin: (Revenue \u2212 COGS) \u00f7 Revenue
\u2022 Cloud MAU: Users with \u22651 xCloud session in trailing 30 days

Caveats & Limitations

\u2022 All data is illustrative and for demonstration purposes only
\u2022 Competitive benchmarks use publicly available estimates
\u2022 Revenue figures include inter-company transfers
\u2022 Cloud gaming metrics exclude preview/beta participants
\u2022 Retention cohorts use intent-to-treat methodology

Refresh Cadence

\u2022 MAU/DAU/Engagement: Daily refresh, T+1 latency
\u2022 Revenue: Weekly refresh (Monday AM), T+3 latency
\u2022 NPS/Sentiment: Quarterly survey, ad-hoc pulse checks
\u2022 Competitive Intel: Monthly update from market research
\u2022 Experiments: Real-time status, weekly metric refresh
